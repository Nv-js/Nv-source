# Nv-source
All components in code and build

> 目录介绍

+ dist/ 

所有组件和样式的输出目录

+ docs/

所有使用文档的目录

+ examples/

组件对应的样例

+ src/js

内置组件和外置组件的目录，其中src/js/outer的目录结构要保证一致，否则会出现报错，当然内部可以自行配置编译脚手架进行使用，每一个外置组件都是一个单独的工程

+ src/less

所有内置组件的样式和对应examples/的展示文档的样式


> 使用方法

直接启动即可，进入对应的examples目录进行查看

```
$ gulp
```
